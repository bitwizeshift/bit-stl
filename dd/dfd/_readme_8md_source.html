11 ## What is `bit::stl`?
12 
13 This is the standard template library for various personal projects of mine.
14 It is both an extension to the C++ standard library, along with a custom implementation
15 of future-proof components (such as c++17 features like std::string_view that is not
16 available to a c++14 compiler).
17 
18 ## What does this library offer?
19 
20 This is a core-library intended as an extension to the standard C++ library.
21 
22 It offers various lightweight/useful utilities that are otherwise missing from the standard library.
23 For example:
24 
25 - Extended type traits that support plural type comparison (`are_same`, `is_derived_from_all`, etc)
26 - GSL suggested types (`span`,`string_span`)
27 - Type-erased types (`array_view`,`map_view`,`set_view`)
28 - EBO generic storage types (`compressed_tuple`,`compressed_pair`)
29 - Dynamic constant types (`dynamic_size_constant`,`dynamic_index_constant`). These allow runtime _or_ compile-time constant values
30 - Circular buffers (`circular_deque`,`circular_queue`,`circular_buffer`)
31 - Concepts! (More on this below)
32 - Many other useful utilities
33 
34 **Note** This is _not_ intended as a replacement for the C++ standard library; just a supplement.
35 
36 ### Concepts
37 
38 This library includes a series of type-traits that validate types for satisfying specific concepts.
39 
40 This is not a true C++ Concepts Light implementation, since it is unable to provide proper diagnostics for which requirements are missed. But it does offer type validation where necessary, allowing for better SFINAE on template types.
41 
42 For example, to test whether a type satisfies a `Container` concept, a trait called `is_container` can be used
43 
44 ### Directory Structuring
45 
46 Although this library does provide C++17 utilities, this primarily acts as a core library extension, and does not follow 1-1 with
47 the standard. Some example differences are:
48 
49 - Composite standard headers are decomposed into individual header files. For example, type trait types are all in individual header files under a `traits` subsystem (unlike the standard's `type_traits` header that contains everything)
50 - The various utility headers are in a `utility` subsystem; likewise, container additions (like `string_view`) are under a `container` subsystem.
51 
52 ### Simple Namespacing
53 
54 The namespace convention is simple; all components reside within `namespace bit::stl`.
55 
56 There are two common `inline namespace`s as well that can be brought in independently; `inline namespace casts` and `inline namespace literals`. The `casts` namespace can be used to bring things like `pointer_cast` or `narrow_cast` to the top-level without
57 requiring polluting the entire global namespace, whereas the `literals` namespace, much like the standard library, brings in
58 any `operator ""` literals.

60 ## Using this library
61 
62 ### Building & Installing
63 
64 Building `bit::stl` is simple, and only requires a CMake version greater than 3.1, and a compatible compiler.
65 
66 Fist, make a directory to build the library in, then build with CMake
67 
68 ```bash
69 mkdir build
70 cd build
71 
72 cmake . -DBIT_STL_BUILD_UNIT_TESTS=Off -DBIT_STL_BUILD_INDEPENDENCE_TESTS=Off # ... any additional toolchain parameters ...
73 cmake --build .
74 ```
75 
76 Omitting the `-DBIT_STL_BUILD_UNIT_TESTS=Off` will build unit tests; likewise `DBIT_STL_BUILD_INDEPENDENCE_TESTS=Off` will
77 build header-independence checks (building each header independently to ensure there are no header-order based transitive dependencies).
78 
79 To install, run:
80 
81 ```bash
82 cmake --build . --target install
83 ```
84 
85 To change the output directory of the installation, you can export `BIT_HOME` to install to a specific directory
86 
87 ```bash
88 export BIT_HOME=/etc/bit
89 cmake --build . --target install
90 ```
91 
92 will install into `/etc/bit/`.
93 
94 ### Through a `cmake` subdirectory
95 
96 Clone/copy/subtree the contents of this repository to a subdirectory, and `add_subdirectory` the directory containing `bit::stl`.
97 
98 To add a dependency to the library, just add `target_link_libraries(<your target> [PUBLIC|INTERFACE|PRIVATE] bit::stl)`
