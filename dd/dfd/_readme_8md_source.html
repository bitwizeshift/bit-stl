11 ## What is `bit::stl`?
12 
13 This is the standard template library for various personal projects of mine.
14 It is both an extension to the C++ standard library, along with a custom implementation
15 of future-proof components (such as c++17 features like std::string_view that is not
16 available to a c++14 compiler).
17 
18 All functions and types are written to replicate the standard namespace's naming
19 conventions for easy use and upgradeability to new c++ library features at is released.
20 
21 **Pedantic Note**: The name `stl` is not meant as a mistaken reference to the c++ standard
22 library, which is often wrongfully referred to as `stl`. It's named this since it truly is
23 a `template`/`inline` library that is used in other projects.
24 
25 ## Namespacing
26 
27 The namespace convention is simple; all components reside within `namespace bit::stl`.
28 
29 There are two common `inline namespace`s as well that can be brought in independently; `inline namespace casts` and `inline namespace literals`. The `casts` namespace can be used to bring things like `pointer_cast` or `narrow_cast` to the top-level without
30 requiring polluting the entire global namespace, whereas the `literals` namespace, much like the standard library, brings in
31 any `operator ""` literals.
