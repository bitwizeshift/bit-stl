Detailed Description

template<typename T>
class bit::stl::clone_ptr< T >

clone_ptr is a smart pointer that owns and manages another object through a pointer and disposes of that object when the clone_ptr goes out of scope

Like unique_ptr, clone_ptr owns unique instances of objects; no two clone_ptrs should ever point to the same instance. Unlike unique_ptr, clone_ptr is copyable – wherein the underlying type's copy-constructor is invoked, and the new object is allocated.

Instead of requiring the Deleter type, this class requires knowledge of both the allocation and deallocation scheme – and thus requires the Allocator itself.

Note
No attempt has been made for this class to support the T[] specialization, since the Allocator concept requires the size of the originally allocated array to perform deallocations. This would require carrying along an extra size argument to properly deallocate, and provides more complications than benefits
Bug:
Currently this marks functions as 'noexcept', when they should be marked 'noexcept(std::is_nothrow_copyable<Allocator>::value)' and 'noexcept(std::is_nothrow_default_constructible<Allocator>::value)'
